# AllControlBase
这是一个通用的控制层模块，目的就是适配不同的平台，通过网络学习于平台无关的数据结构来使用这个通用的控制层模块，来适配不同的平台
具体需求拆解如下：

架构逻辑：

前端（网络层）：由“主体导航架构（Backbone）” + “平台特异性输出头（Head）”组成。

后端（控制层）：这是一个通用控制器，它是一套固定的代码/逻辑框架。

部署流程：当从无人车切换到无人机时，Backbone 不变，通用控制器代码不变，只更换并微调那个小小的“输出头（Head）”。

核心痛点（待解决的问题）：

通用控制器的设计：它不依赖视觉感知（那是网络的事），它只负责接收指令并把机器“开”起来。需要设计它的内部逻辑，使其能用同一套逻辑兼容车、无人机等不同物理特性的平台。

接口设计（Head 输出什么）：需要定义一种通用的数据格式，作为网络 Head 的输出，同时也是通用控制器的输入。这个数据必须足够通用，能被车理解，也能被无人机理解。
